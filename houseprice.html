Data Science and ML: House Price Prediction

Project Overview

This project shows how modern Data Science and Machine Learning techniques can be applied for solving complex tasks, such as the prediction of a house prices based on descriptive features.

The dataset I worked on for this project contains information about nearly 1500 house deals. Each deal includes the final house price along with 80 additional descriptive parameters that provide detailed information about the property and the transaction.

GOAL

The main objective is to predict house prices based on the descriptive parameters with highest possible accuracy. The secondary goal is to find the most influential parameters, which affect the end price.

METHOD

The project is structured according to the Data Science Lifecycle. Advanced data science and machine learning techniques were applied in order to reach objectives and train the predicting model. The code is written in Python.

RESULT

As a result the best-performing prediction model was selected. This model allows to accurately predict the house price based on certain input parameters. In addition, the list of parameters that are most correlated to the house price to the house price was created.

Exploring the data is a crucial step as it provides insights into correlations between different variables and helps uncover valuable information about the business. For example, in the case of house sales, it was discovered that there is a seasonal pattern in the number of houses sold, with the highest number of deals occurring during the summer season.

Feature Engineering

In this step, features to be used for training the predictive model are selected. The purpose of removing features is to prevent overfitting, which can lead to lower accuracy. Therefore, only features that have significance are kept.

I used an automated method, called step-wise selection, which returns the list of best features:

As a result, only 36 out of 96 features were selected.

Hyperparameter Tuning

Before training the predictive model, the tuning of its main parameters must be done. To determine the values to which these parameters should be adjusted, I used a cross-validator's exhaustive search. This method is computationally intensive but ensures the selection of the best parameter combination.

Hyperparameter tuning is performed for each individual model. An example for Gradient Booster is shown:

Received sets of hyperparameters will be used to adjust house price predictive models.

Training the model

I selected five machine learning models and compared their performance based on accuracy for this project. The tested models include Linear Regression, K-Neighbors Regressor, Support Vector Regressor, Ridge Regression, and Gradient Booster.

The most accurate model was the Support Vector Regressor (SVR), with the Gradient Booster (GB) coming in a close second. Here is what the training of the model looks like, using the example of SVR:

The trained SVR model can predict the final sale price of a house with an impressive average accuracy of 92%, while the maximum accuracy achieved in a good train/test split reaches 95%.

Taking into account the irreducible error, assosiated, for example, with house price variability on the market, the achieved accuracy of SVR predicting model is close to perfect.
